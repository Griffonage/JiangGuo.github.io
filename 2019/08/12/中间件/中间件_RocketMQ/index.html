修改配置文件(防止内存不足)

修改启动配置参数

vim bin/runserver.sh

修改部分如下图所示

同理，修改几个bin目录下的文件

runbroker.sh

修改67行为 JAVA_OPT="${JAVA_OPT} -server -Xms128m -Xmx128m -Xmn256"

tools.sh

修改39行JAVA_OPT="${JAVA_OPT} -server -Xms128 -Xmx128 -Xmn64m -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=128m"

运行

  • 运行 name server
    • 启动:nohup sh bin/mqnamesrv &
    • 查看启动日志:tail -f ~/logs/rocketmqlogs/namesrv.log
  • 运行broker
    • 启动:nohup sh bin/mqbroker -n 120.26.173.108:9876 &
    • 查看启动日志:tail -f ~/logs/rocketmqlogs/broker.log
  • 关闭服务命令
    • sh bin/mqshutdown broker
    • sh bin/mqshutdown namesrv
  • 注：要想完全清空数据，删除文件夹~/store，然后重启

安装控制台

1，下载：

​ git clone https://github.com/apache/rocketmq-externals.git

2，找到rocketmq-console/src/main/resources/application.properties 根据需求，修改配置

​ server.port=8081

​ # name server地址

​ # 也可以不修改，在启动完console后，在控制台导航栏 - 运维 - NameSvrAddrList一栏设置

​ rocketmq.config.namesrvAddr=localhost:9876

3，修改 pom.xml ，修改RocketMQ相关依赖的版本

​ <rocketmq.version>4.7.1</rocketmq.version>

4，切换到控制台目录 cd rocketmq-console

​ mvn clean package -DskipTests
